RAASA practical packet reproduction commands

Assumption:
- run from the repository root
- dependencies are installed from environment/requirements.txt
- SSH keys are kept outside the repository; either pass -KeyPath explicitly or set RAASA_AWS_KEY_PATH

1. Local adaptive run
python -m raasa.experiments.run_experiment --mode raasa --scenario small_tuned --run-id practical_small_linear --config raasa/configs/config_tuned_small_linear.yaml

2. Static L1 baseline
python -m raasa.experiments.run_experiment --mode static_L1 --scenario small_tuned --run-id practical_static_l1 --config raasa/configs/config_tuned_small_linear.yaml

3. Static L3 baseline
python -m raasa.experiments.run_experiment --mode static_L3 --scenario small_tuned --run-id practical_static_l3 --config raasa/configs/config_tuned_small_linear.yaml

4. Medium adaptive run
python -m raasa.experiments.run_experiment --mode raasa --scenario medium --run-id practical_medium_linear --config raasa/configs/config_tuned_small_linear.yaml

5. Large adaptive run
python -m raasa.experiments.run_experiment --mode raasa --scenario large --run-id practical_large_linear --config raasa/configs/config_tuned_small_linear.yaml

6. Write a summary from a JSONL log
python -c "from raasa.analysis.metrics import write_metrics_summary; write_metrics_summary('raasa/logs/run_example.jsonl')"

7. Run the unit tests
pytest -q

8. AWS/K8s bootstrap path
bash bootstrap_k8s_ebpf.sh

9. K8s adaptive run after bootstrap
python -m raasa.experiments.run_experiment --mode raasa --scenario small_tuned --backend k8s --run-id practical_k8s_r1 --config raasa/configs/config.yaml

10. Collect live AWS validation evidence from a running K3s node
powershell -ExecutionPolicy Bypass -File raasa/scripts/collect_aws_live_validation.ps1 -TargetHost <AWS_IPV4> -KeyPath $env:RAASA_AWS_KEY_PATH

11. Run the Phase 1B blast-radius benchmark against a live K3s node
powershell -ExecutionPolicy Bypass -File raasa/scripts/run_phase1b_blast_radius.ps1 -TargetHost <AWS_IPV4> -KeyPath $env:RAASA_AWS_KEY_PATH

12. Deploy a fresh RAASA agent image tag to the live K3s node
powershell -ExecutionPolicy Bypass -File raasa/scripts/deploy_agent_image_to_aws.ps1 -TargetHost <AWS_IPV4> -KeyPath $env:RAASA_AWS_KEY_PATH -ImageTag raasa/agent:<TAG>

13. Run the Phase 1C pod-specific enforcement validation
powershell -ExecutionPolicy Bypass -File raasa/scripts/run_phase1c_pod_specific_validation.ps1 -TargetHost <AWS_IPV4> -KeyPath $env:RAASA_AWS_KEY_PATH

14. Run the Phase 1D universal pod-resolution validation
powershell -ExecutionPolicy Bypass -File raasa/scripts/run_phase1d_resolution_validation.ps1 -TargetHost <AWS_IPV4> -KeyPath $env:RAASA_AWS_KEY_PATH
